Guide the user through installing or switching an Aegis statusline preset for Claude Code. This is a Claude Code slash command — work entirely in-host. **IMPORTANT: Aegis never runs a CLI to mutate `settings.json`. This command guides you (the model) and the user to make the change in-host; there is no `aegis` binary doing the write.**

The per-preset `.mjs` scripts and `.settings.json.snippet` files are **generated by `scripts/project.mjs`** into `adapters/claude/statuslines/`. They are downstream artifacts of the canonical presets under `statuslines/` — do not hand-edit them.

## 1. Pick a preset

If the user passed a preset name as an argument and it matches one below, use it. Otherwise ask with `AskUserQuestion`. The 8 presets are:

- `minimal-mono` — safe default, no colour.
- `essential` — activity-focused.
- `tokyo-night` — truecolor palette.
- `gruvbox-dark` — retro-warm palette.
- `git-forward` — branch + PR badge.
- `cost-aware` — session cost / duration.
- `token-budget` — token transparency.
- `verbose-hud` — everything.

## 2. Preview the rendered output

Pipe a sample stdin fixture (a representative Claude Code statusline JSON payload — model, cwd, cost, token counts) into the projected script and show the rendered line:

```
echo '<sample-statusline-stdin-json>' | node adapters/claude/statuslines/<preset>.mjs
```

Show the user the rendered output so they can confirm the look before committing. If the script errors or emits nothing, stop and report rather than proceeding to merge.

## 3. Inspect and back up existing settings

Read the user's `~/.claude/settings.json`. Detect any existing `statusLine` block and identify its origin:

- **claude-hud** — a claude-hud-managed block.
- **cc-statusline** — a cc-statusline-managed block.
- **custom** — a hand-written `statusLine` the user owns.
- **none** — no `statusLine` key present.

If a `statusLine` block exists, back the file up first (e.g. copy to `~/.claude/settings.json.bak-<timestamp>`) and explain to the user exactly what will be replaced and why.

## 4. Merge the preset block (no BOM)

Read the preset's `statusLine` block from `adapters/claude/statuslines/<preset>.settings.json.snippet` and merge it into `~/.claude/settings.json`, replacing any existing `statusLine` key. Preserve all other keys. Write the file as UTF-8 **without a BOM** — a leading BOM will break Claude Code's JSON parse and silently disable the statusline.

## 5. Restart guidance + troubleshooting

Tell the user to restart Claude Code (or start a fresh session) for the statusline to take effect. If the statusline does not appear, walk the disappearing-statusline checklist:

- The script must **exit 0** — a non-zero exit makes Claude Code hide the statusline.
- The script must produce **non-empty stdout** — an empty line is treated as no statusline.
- Output must be **sanitized** — no stray BOM, no unescaped control characters; truecolor/ANSI only where the terminal supports it (fall back to `minimal-mono` if colour renders as raw escape codes).

Confirm the final `statusLine` block in `settings.json` points at the projected `adapters/claude/statuslines/<preset>.mjs` path before declaring done.
