## Locked Decisions

- **`aegis-` prefix on every projected skill name** to dodge collisions with Codex reserved names (`default`, `worker`, `explorer`) and across plugins. Reserved-name guard hard-fails at projection time.
- **Agent-as-skill folding** ŌĆö `agents/<name>.md` projects to `.codex/plugins/aegis/skills/aegis-<name>/SKILL.md`. Codex *does* have a native subagent primitive (`.codex/agents/*.toml`), but it is **not plugin-distributable** (D-02, verified 2026-06-20), so a plugin-shipped Aegis cannot deliver it ŌĆö folding is the viable path. Agents become discoverable skills users invoke explicitly (runtime-verified: the install test's skill list included agent-derived skills). They lose dispatchable-subagent semantics on Codex.
- **Command-as-dispatcher pattern** ŌĆö `commands/<name>.md` projects to a thin `SKILL.md` (~10ŌĆō15 line body) referencing the canonical skill/workflow path. Skill+command name collisions resolve with a `__command` suffix on the command-derived skill.
- **Plugin root at `.codex/plugins/aegis/`** ŌĆö self-contained plugin directory; manifest at `.codex-plugin/plugin.json` inside it, skills at `skills/`, MCP stub at `.mcp.json`. This is the official Codex plugin layout verified against codex 0.141.0 (D-04).
- **Object source form in marketplace** ŌĆö `{source:"local", path:"./.codex/plugins/aegis"}` required for `codex plugin add` to resolve the plugin. The string `"./"` form is broken (D-05).

## Statuslines

No documented Codex statusline. The bottom row of the Codex TUI is reserved for keybind hints and is not a user-scriptable status-row slot. Aegis ships statusline presets at v0.0.4 (Claude Code first-class); Codex stays an honest gap until Codex grows a documented statusline surface.

## Dynamic workflows (gap)

Claude Code's dynamic workflows are a host-resident built-in (`Workflow` tool): the model writes a JS orchestration script and a background runtime fans it out across dozens-to-hundreds of subagents, saving reusable scripts to `.claude/workflows/`. This is not a plugin extension point ŌĆö Aegis cannot ship, declare, or project it, and no equivalent exists on this host. The portable substitute is the `orchestration` skill (Ōēż5-wave `Task()` fan-out with in-session synthesis), which runs identically everywhere but does not reach the hundreds-of-agents, context-isolated, resumable regime. For very large audits/migrations on this host, the Ōēż5-wave skill is the ceiling.

## Permissions

No projection. Codex has **no host primitive for agent-level tool restriction** ŌĆö
agents are folded into the skills tree (no distinct subagent primitive), and there
is no per-agent `tools` allowlist or permission block to emit into. Aegis agent
permissions are therefore **advisory** on Codex: `manifest/permissions.json`
documents the intended posture, but nothing enforces it at runtime here. Per
silent-failure discipline, Aegis does not write enforcement prose into
`.codex-plugin/AGENTS.md` that the host won't honour. (Decision D6.)

- **Source of truth:** `manifest/permissions.json` (the buckets, per-agent grants,
  and the plugin-level cross-cutting deny).
- **Author guide:** `docs/agent-permissions.md`.
- **Enforced hosts:** Claude (`tools:` allowlist) and OpenCode
  (`agent.<name>.permission`) ŌĆö see those `adapters/<host>/projection.md`.

## Generator

`scripts/project.mjs` ŌåÆ `projectCodex()`. Sub-functions:

- `projectCodexSkills()` ŌĆö walks `skills/core/`, `skills/languages/`, `skills/workflows/`.
- `projectCodexAgentsAsSkills()` ŌĆö walks `agents/*.md`.
- `projectCodexCommandsAsDispatchers()` ŌĆö walks `commands/*.md`; applies `__command` suffix on collision.
- `projectCodexRules()` ŌĆö concatenates `rules/*.md` into `.codex-plugin/AGENTS.md`.

Helpers: `prefixAegis(name)` for collision-avoidance prefixing, `extractIronLawsBlock()` for the rules surface, plus the `atomicReplace()` idempotency primitive. Reserved-name and name-shape guards are enforced inline via the `CODEX_RESERVED_NAMES` constant and `CODEX_NAME_PATTERN` regex (no dedicated helper function). Dispatcher bodies are composed inline inside `projectCodexCommandsAsDispatchers()`. Idempotent: re-running produces identical output.

- `projectCodexHooks(intents)` (v0.2.1, AG-0239) ŌĆö emits `.codex/plugins/aegis/hooks/hooks.json` in Codex matcher-group shape for the 4 mapped events; bundles the Claude hook scripts (version-stamped, executable) into `.codex/plugins/aegis/hooks/`; bundles `manifest/permissions.json` ŌåÆ `.codex/plugins/aegis/hooks/permissions.json` for script-relative deny-config resolution. Called from `projectCodex(hookIntents)`. Idempotent.
