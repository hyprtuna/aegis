# Statusline

Aegis ships **8 statusline presets for Claude Code**, rendered by a bulletproof
Node runtime. No dependencies, no build step — just Node 18+ built-ins and plain
`.mjs` ESM. A statusline is the bar Claude Code runs after each assistant turn,
fed a JSON session payload on stdin.

> **Host support.** Claude Code is the only host with a real statusline today.
> OpenCode, Codex, Cursor, and Zed have no native statusline (and no
> `subagentStatusLine`) equivalent — these are tracked as honest gaps in
> `adapters/<host>/projection.md`, not silently dropped. The preset descriptors
> stay host-agnostic so a future hook can render them elsewhere.

## Install flow

Run the slash command and follow the prompts:

```
/aegis:statusline             # asks which preset to install
/aegis:statusline git-forward # installs a named preset directly
```

The command walks you through four steps:

1. **Pick** a preset (from the 8 above).
2. **Preview** the rendered output — a sample stdin payload is piped through the
   preset's projected script so you can confirm the look before committing:
   ```bash
   echo '<sample-statusline-stdin-json>' | node adapters/claude/statuslines/<preset>.mjs
   ```
3. **Back up** your existing `~/.claude/settings.json` if it already has a
   `statusLine` block (claude-hud, cc-statusline, or hand-written).
4. **Merge** the preset's `statusLine` block into `~/.claude/settings.json`,
   replacing any existing `statusLine` key and preserving all other keys.

The merged block looks like this:

```json
{
  "statusLine": {
    "type": "command",
    "command": "node \"${CLAUDE_PLUGIN_ROOT}/adapters/claude/statuslines/<preset>.mjs\"",
    "padding": 0
  }
}
```

Write the file as **UTF-8 without a BOM** — a leading BOM breaks Claude Code's
JSON parse and silently disables the statusline. Restart Claude Code (or start a
fresh session) for the change to take effect.

> **Where these files come from.** The per-preset `<preset>.mjs` scripts and
> `<preset>.settings.json.snippet` blocks under `adapters/claude/statuslines/`
> are **generated by `scripts/project.mjs`** from the canonical presets under
> `statuslines/`. They are downstream artifacts — do not hand-edit them, and
> Aegis never runs a CLI to mutate `settings.json`; the merge is done in-host.

## Switching and customizing

- **Switch presets** — just re-run `/aegis:statusline` and pick a different one.
  It replaces the `statusLine` block in place.
- **Themes** live in `statuslines/_shared/themes/` (`mono`, `default`,
  `tokyo-night`, `gruvbox-dark`). Color values are named-ANSI, a 256-color
  integer, or `#rrggbb` hex. Themes are additive — more can ship between
  releases without a schema bump.
- **Segments are composable.** Each preset descriptor's `segments` is an array
  of *lines*; each line is an ordered array of segment-ID strings. Available
  segments include `model`, `project`, `git`, `pr`, `effort`, `context`,
  `context-detailed`, `usage`, `prompt-cache`, `cost`, `tools`, `agents`,
  `todos`, and `session-time`.
- **Refresh intervals apply only to time-based presets.** Event-only presets
  (`minimal-mono`, `essential`, `token-budget`) redraw on each turn. Presets
  with countdowns set a `refreshInterval`: `tokyo-night` / `gruvbox-dark` at
  30s, `cost-aware` / `verbose-hud` at 15s, `git-forward` at 10s.

## Troubleshooting — the disappearing-statusline checklist

Claude Code blanks the statusline on a **non-zero exit, empty stdout, or a slow
script**. The Aegis runtime is built to never trip those:

- it always **exits 0** (rendering is wrapped in try/catch with
  `process.exit(0)` in a `finally`);
- it always prints **at least `[Aegis]`** — never wholly empty stdout;
- it **sanitizes** every interpolated stdin string (strips C0 control chars,
  keeps the ESC introducer so ANSI/OSC-8 still work) and writes nothing to
  stderr;
- it reads stdin with a 400ms timeout, and on JSON parse failure emits `[Aegis]`
  and exits 0;
- it **defaults `COLUMNS` to 120** when width is unknown;
- a broken individual segment is dropped by `compose()` rather than taking down
  the whole bar.

If the row is still blank, work through this:

1. **Check the command path** in `settings.json` points at the projected
   `adapters/claude/statuslines/<preset>.mjs`.
2. **Check `node` is on PATH** in the environment Claude Code launches.
3. **Run the command manually**, piping a sample stdin JSON, and inspect the
   output:
   ```bash
   echo '<sample-statusline-stdin-json>' | node adapters/claude/statuslines/<preset>.mjs
   ```
4. **Check `settings.json` has no BOM** — re-save as UTF-8 without one.
5. **Colour rendering as raw escape codes?** Your terminal lacks truecolor —
   fall back to `minimal-mono`.

> **Subagent variant.** A separate `subagentStatusLine` runtime
> (`_subagent.mjs`) exists for the subagent bar; its JSON contract is documented
> in `statuslines/_shared/subagent-contract.md`.
